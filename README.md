# ASL Static Handshape Recognition — Research Implementation

Fine-grained recognition of visually confusable American Sign Language static
handshapes via skeletal vectorisation and deep sequence modelling.

> Companion code for the paper:  
> *"Fine-Grained Recognition of Visually Confusable ASL Static Handshapes via
> Skeletal Vectorisation and Deep Sequence Modelling"*  
> L T D B Fernando, L D M P Fernando

---

### Step 1 — Download the Dataset

The pipeline uses the **Google ASL Fingerspelling Recognition** dataset
(Kaggle 2023).  It provides pre-labelled video sequences of fingerspelled
letters from ~100 Deaf signers, with MediaPipe landmarks already extracted
per frame in Parquet format.

#### 1a. Install the Kaggle CLI and authenticate

```bash
pip install kaggle
```

Place your `kaggle.json` API token at `~/.kaggle/kaggle.json`
(download from https://www.kaggle.com/settings → API → Create New Token).

#### 1b. Download and extract

```bash
kaggle competitions download -c asl-fingerspelling -p data/raw
cd data/raw
unzip asl-fingerspelling.zip
```

After extraction `data/raw/` should contain:

```
data/raw/
├── train.csv
├── supplemental_metadata.csv
└── train_landmarks/
    ├── 1000035161.parquet
    ├── 1000106739.parquet
    └── ...   (~67 000 parquet files)
```

### Step 2 — Preprocessing (Skeleton Extraction + Vectorisation)

Run this once.  It:
1. Filters `train.csv` to **single-character static-letter sequences** (24 classes).
2. Loads each parquet → extracts dominant hand landmarks (21 keypoints × 3 coords).
3. Normalises every sequence: translate to wrist origin → scale → mirror left hand.
4. Resamples to fixed length T = 30 frames.
5. Saves stratified train / val / test splits as compressed `.npz` files.

```bash
python -c "
from src.data_loader import build_raw_index, load_all_sequences
from src.preprocess  import build_processed_dataset

index_df              = build_raw_index()
sequences, labels, lf = load_all_sequences(index_df)
build_processed_dataset(sequences, labels, lf)
"
```

Expected output:
```
[data_loader] Found 43 812 single-letter sequences across 24 classes.
[data_loader] Loaded 43 597 sequences (215 skipped).
[preprocess]  Saved train: 33 136 samples → data/processed/train.npz
[preprocess]  Saved val:   5 227 samples  → data/processed/val.npz
[preprocess]  Saved test:  5 234 samples  → data/processed/test.npz
```

The processed data is stored in:
```
data/processed/
├── train.npz   X: (33136, 30, 21, 3)  y: (33136,)
├── val.npz     X: ( 5227, 30, 21, 3)  y: ( 5227,)
└── test.npz    X: ( 5234, 30, 21, 3)  y: ( 5234,)
```

### Step 3 — Data Augmentation

Augmentation is applied **on-the-fly** during training via `SkeletonAugmenter`
inside `ASLDataset`.  No separate step is needed.

The six operations and their parameters (all configured in `config.py`):

| # | Operation | Parameter | Applied to |
|---|-----------|-----------|------------|
| 1 | Mirror flip | negate x | All (p=0.5) |
| 2 | In-plane rotation | ±15° | All (p=0.5), hard clusters (p=0.8) |
| 3 | Scale jitter | ×[0.85, 1.15] | All (p=0.5), hard clusters (p=0.8) |
| 4 | Speed jitter | ×[0.80, 1.20] | All (p=0.5) |
| 5 | Gaussian noise | σ=0.02 | All (always) |
| 6 | MixSkel | λ∈[0.3, 0.7] intra-class | All (p=0.5), hard clusters (p=0.8) |

Hard clusters (higher augmentation probability):
- `{A, S, E, T, N, M}` — closed-fist confusables
- `{K, R, U, V, W}` — extended-finger confusables

### Step 4 — Train a Model

Five architectures are available:

| ID | Flag | Architecture | SupCon loss |
|----|------|-------------|-------------|
| M1 | `bilstm` | Bidirectional LSTM | No |
| M2 | `tcn` | Temporal Convolutional Network | No |
| M3 | `transformer` | Transformer Encoder | No |
| M4 | `stgcn` | ST-GCN (spatial-temporal graph) | Yes |
| M5 | `gat` | Graph-Attention Transformer | Yes |

Train the recommended model (M5):

```bash
python src/train.py --model gat --epochs 100
```

Train all models for comparison:

```bash
for model in bilstm tcn transformer stgcn gat; do
    python src/train.py --model $model --epochs 100
done
```

Checkpoints are saved to `checkpoints/<model>_best.pt`.  
Training logs (CSV) are saved to `logs/<model>_log.csv`.

#### Training hyperparameters (edit `config.py` to change):

```python
BATCH_SIZE    = 64
NUM_EPOCHS    = 100
LR            = 1e-3       # cosine-annealed to LR_MIN = 1e-5
WEIGHT_DECAY  = 1e-4
PATIENCE      = 15         # early stopping
LABEL_SMOOTH  = 0.10
SUPCON_LAMBDA = 0.30       # weight of contrastive loss (M4, M5 only)
```

### Step 5 — Evaluate

python src/evaluate.py --model gat

## Understanding the Architecture

### Why graph-based models (M4, M5)?

Standard flat models (M1–M3) treat the 63 keypoint coordinates as an
unstructured vector, applying the same operation uniformly to all coordinates.
They cannot reason about which *finger* is involved.

M4 (ST-GCN) and M5 (GAT-Transformer) represent the hand as a **graph**:
- 21 nodes = hand keypoints
- Edges = anatomical bone connections

Graph convolution propagates information only along connected joints.
This encodes *finger-level topology* — the exact structural level at which
confusable letters differ (e.g., A vs S differs only in thumb placement
relative to the closed fist).

### Why the SupCon loss for M4 and M5?

Supervised Contrastive Loss with hard-pair mining pushes the embeddings of
confusable-cluster letters (e.g., M and N) *apart* in representation space,
even when overall accuracy is already high.  It is only applied to M4/M5
because those models produce finger-topology-aware embeddings worth contrasting.
